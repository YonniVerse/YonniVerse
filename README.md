## 🧩 How I Think About Problems

```
A problem is just a question waiting for the right framing.

  1. Understand before you touch the keyboard.
  2. Break it down until each piece is obvious.
  3. Write the simplest thing that works.
  4. Make it fast only when slowness hurts someone.
  5. Read the output like a user, not like the author.

Complexity is the enemy. Clarity is the goal.
Emotion is the result — when you get it right.
```
