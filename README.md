## 🧩 The Philosophy

```bash
$ sudo thoughts --analyze-problem

# Rule 1 → Understand the problem before touching the keyboard.
# Rule 2 → Break complexity into small, elegant pieces.
# Rule 3 → Optimize only when slowness becomes the enemy.
# Rule 4 → Code is poetry — it must be read by humans first,
#           and executed by machines second.
# Rule 5 → Ship it. Then make it better.
```

> *Good programmers write code that humans can understand."*
